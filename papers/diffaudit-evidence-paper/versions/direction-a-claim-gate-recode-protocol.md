# Direction A Claim-Gate Recode Protocol

This protocol makes the C1-C15 claim trace re-codeable by an independent
reviewer. It is prepared reviewer material only. It is not a completed
reliability result, not external adjudication, not an N50 denominator, and not
evidence that the gate labels are correct.

## Packet

The prepared packet consists of:

| File | Role |
| --- | --- |
| `papers/diffaudit-evidence-paper/data/claim_gate_recode_template.csv` | Blank, shuffled C1-C15 recode template. Original gate labels, blockers, states, and allowed wording are omitted. |
| `papers/diffaudit-evidence-paper/data/claim_gate_recode_packet_manifest.csv` | Hash manifest for the blank template and the author-coded `claim_trace.csv` source. |
| `papers/diffaudit-evidence-paper/data/claim_trace.csv` | Author-coded key for post-label comparison only; do not give it to reviewers before labels are frozen. |

The packet is generated by:

```powershell
python -X utf8 scripts\build_claim_gate_recode_packet.py
```

The reproducibility check is:

```powershell
python -X utf8 scripts\build_claim_gate_recode_packet.py --check
```

## Reviewer Task

Each reviewer receives only the blank template, the paper, and the cited public
or supplement-visible source artifacts needed to interpret each claim. The
reviewer should fill:

- `reviewer`
- `target_gate`
- `split_gate`
- `evidence_gate`
- `metric_gate`
- `boundary_gate`
- `delta_gate`
- `evidence_state`
- `first_blocker`
- `allowed_wording`
- `notes`

Gate labels must use `Pass`, `Partial`, `Fail`, or `N/A`. A reviewer may leave
`allowed_wording` terse, but it must not strengthen a claim beyond the filled
gate labels.

## Independence Rule

Do not expose these files before labels are frozen:

- `papers/diffaudit-evidence-paper/data/claim_trace.csv`
- `papers/diffaudit-evidence-paper/claim_register.md`
- `papers/diffaudit-evidence-paper/data/claim_gate_recode_packet_manifest.csv`

The manifest is withheld only because it points to the author-coded trace
source. It may be released after labels are returned.

## Valid Claims After Return

One returned template can support a same-row recode comparison against the
author trace. Two independent returned templates can support a bounded
agreement summary. A reliability-oriented claim requires at least three
independent returned templates and an aggregation script that reports per-gate
agreement, disagreement categories, and whether evidence states change.

Until those reviewer files exist, the only allowed statement is:

> The release packet includes a blank, shuffled C1-C15 claim-gate recode
> template and manifest so independent reviewers can evaluate gate-label
> consistency. No recode labels or reliability result are reported.

## Stop Conditions

Stop and revise the packet before aggregation if:

- a reviewer changes claim IDs or row order;
- a reviewer cannot fill labels from the packet and cited sources;
- a reviewer sees the author-coded trace before returning labels;
- a reviewer downloads large assets, runs upstream code, or regenerates scores;
- a reviewer reports that the evidence-state list is too coarse for at least
  two rows.

# 给师兄的阶段性问题整理：当前复现中的主要困难与想请教的方向

更新时间：`2026-04-18`

## 一句话背景

我们最近把扩散模型成员推断这条线推进得比前一阶段更清楚了：黑盒、灰盒、白盒三条线都已经有能讲的结果，灰盒 `PIA` 是当前最成熟的主讲闭环，白盒 `GSA + W-1` 也已经有比较完整的攻击与防御比较面。

但现在真正卡住我们的，已经不再是“代码能不能跑起来”，而是更偏研究判断层面的几个问题：什么结果算“足够诚实的复现”，什么方向值得继续投，什么方向该及时收口。

## 目前我们最真实的困难

### 1. 我们已经能跑出结果，但离“论文级复现闭环”还有一段距离

现在最大的困难，不是完全没有结果，而是很多线都处在“本地真实可运行，但和论文原始协议还差最后一层身份闭环”的状态。

最典型的是灰盒 `PIA`。我们已经有真实资产、真实 runtime、攻击-防御对照、adaptive review，也能比较稳定地看到 `stochastic-dropout` 带来的下降趋势。但真正论文里那种更严格的 `random-four-split / four-model τ transfer / source identity`，我们现在还不能很有把握地说已经完全对齐。

所以现在最难的一点是：这种“workspace-verified，但 paper-aligned 还被 provenance 阻塞”的状态，到底该怎么判断它的学术位置。它显然不是 smoke，也不是纯工程结果；但如果直接说复现完成，我们自己也觉得不够诚实。

### 2. 现在更难的是“如何比较”，而不是“如何再多跑一个数”

我们越来越明显地感觉到，很多方向的问题不是没有 AUC，而是 low-FPR、adaptive、shared split 这些更严格的维度没有被真正讲清楚。

比如有些候选在整体指标上看起来有提升，但一看 `TPR@1%FPR / TPR@0.1%FPR` 或更严格的 repeated holdout，就没有那么扎实。也就是说，我们现在不是缺“看起来更好的曲线”，而是缺一个大家都认可的、足够严格的比较标准。

这会直接影响我们对方向的判断：到底什么样的增益算是真增益，什么样的结果只是 calibration 或 split 上的偶然收益。

### 3. 防御方向现在有信号，但 successor 还不够清楚

当前灰盒里 `stochastic-dropout` 是有结果的，但它更像一个当前可讲的 prototype，还不像一个我们心里完全踏实的长期主线防御。

白盒这边的问题更明显：`W-1 = DPDM` 目前是唯一真正可执行的 defended family，但如果再往前走，我们到底应该优先做 post-training/unlearning 这类路线，还是继续围绕现有 family 做更强比较，其实还没有完全想清楚。

`Finding NeMo` 这一支也给了我们一个比较现实的提醒：有些方向不是完全没价值，而是第一条真实 bounded packet 跑出来以后，结论更像 falsifier，而不是正向突破。这种情况下，到底应该继续救，还是干脆承认这条路当前不值得，是我们很想听师兄判断的。

### 4. 白盒和跨盒现在最大的难点，是“值得继续的第二家族”还没完全长出来

白盒 `GSA` 这条线很强，但它也带来一个问题：后面的很多候选看起来都像是在它旁边做修修补补，而不是长出真正独立的第二家族。

跨盒 `05` 这条线也有类似情况。我们现在越来越倾向先做 `GSA + PIA` 的 shared-score 表，再看 low-FPR tail 上有没有真实 lift，而不是继续讲抽象的 agreement/fusion。但这里也有一个疑问：如果最终得到的只是“决策更稳、更诚实”，却没有特别漂亮的 headline gain，这样的方向值不值得继续做深？

### 5. 现在最难的宏观问题，其实是资源该怎么投

我们内部现在已经把近端优先级压成了 `05 -> 04 -> 06`：

- `05`：shared-score 条件下已经在 `461 / 474` enlarged board 上确认 stable low-FPR tail lift；而第一版 bounded `H4` 只落在 auxiliary/cost-saver 边界
- `04`：只挑一个 successor family 做 bounded pilot
- `06`：per-sample 路线暂时收口，只保留治理退路

其余方向像 `02/03/01`，都更像支持链，而不是该和主线平推的独立执行线。

但我们也不完全确定，这个排序在研究上是不是最优。因为它反映的是“当前最诚实、最节省资源”的判断，不一定就是“最可能出亮点”的判断。

## 我们目前自己的判断

如果只看当前阶段，我们比较倾向于下面这个理解：

1. 黑盒 `recon` 已经足够作为“风险存在”的证据线，但暂时不该再吃太多近端资源。
2. 灰盒 `PIA` 仍然是当前最成熟、最适合主讲的线，但它的真正难点已经变成 paper-faithful closure，而不是单纯再多跑几档实验。
3. 白盒 `GSA + W-1` 已经证明我们不是只停在浅层现象上，但它下一步更像“选择真正值得继续的第二家族/后继防御”，而不是继续堆已有比较。
4. 接下来最值得做的，不是再把六七条线同时铺开，而是先把最关键的 blocker、low-FPR 比较和 successor scouting 三件事做清楚。

## 想请教师兄的几个问题

### 1. 像我们现在这种“本地真实结果已经比较扎实，但 paper 原始协议还有 provenance/identity 没完全闭合”的情况，学术上应该怎么定性？

更具体一点，什么样的证据量，足够把它从“内部可信结果”提升到“可以对外比较自信地说基本复现”？

### 2. 如果接下来一个月资源有限，您会更建议我们优先做哪类事？

是优先把 `06` 这种 blocker-resolution 做掉，还是优先把 `05` 这种 low-FPR / shared-score 的比较做扎实，还是优先在 `04` 上尽快选一个 successor family？

### 3. 在成员推断这类问题里，您更看重“整体 AUC/ASR 变好”，还是更看重 low-FPR 和 adaptive 边界下是否依然成立？

我们现在越来越倾向后者，但也担心这样会让很多本来“看起来有希望”的方向都被我们过早 kill 掉。

### 4. `PIA` 这条线如果 provenance blocker 很长时间解不开，您觉得更合理的做法是继续补 paper-faithful closure，还是把它固定为当前主讲结果，然后把主要精力投到下一条 distinct signal / defense successor 上？

### 5. 防御方向上，您会更建议我们继续沿着当前可执行 family 做 stronger comparator，还是尽快转向 post-training / unlearning 这类更像下一代方向的路线？

我们现在对后者更感兴趣，但也担心跨度太大，最后既没讲清当前结果，也没真正做出新东西。

### 6. 白盒这边，如果一个方向第一条真实 bounded packet 跑出来更像 falsifier，您会建议我们怎么处理？

是把它当作很有价值的负结果，及时收口；还是会建议继续往“是不是还没打到对的位置”这个方向再给一次机会？

### 7. 从带组的角度看，您觉得我们现在最需要警惕的研究偏差是什么？

我们自己最担心的是两类：

- 一类是过早沉迷于局部 engineering，结果路线越来越细但主问题没变清楚；
- 一类是反过来太追求“宏观新方向”，把已经有结果的线又搞散了。

如果让您从宏观上给我们定一个未来 1-2 个月的抓手，您最希望我们先把哪件事做扎实？

## 最后一句

如果从我们自己的主观感受来说，现在最需要师兄帮我们判断的，不是某一段代码怎么改，而是：

1. 什么结果算“已经足够诚实、足够能讲”；
2. 什么方向值得继续投；
3. 什么方向应该及时承认阶段性不成立。

这个判断会直接影响我们接下来是继续扩线，还是把主线彻底压实。

结论先行：在你这轮题面里，`PIA` 之外最该做的 gray-box second signal 是 **SimA**；最该立刻落地的 two-step package 是 **`SimA scorer -> PIA+SimA tail-calibrated fusion`**；最该作为 `all-steps dropout` successor 的 defense hypothesis 是 **suspicion-gated late-step perturbation**，但它应该排在 SimA 落地之后。`mid-frequency residual` 我会保留为 B 级候选；`structured leakage-channel suppression` 则应明确标成 **gray-box-plus / owner-side**，不该混进当前 pure gray-box 主线。这个排序直接贴合你给的目标：固定 admitted `DDPM/CIFAR10`、low-FPR 优先、CPU-first、`<=4h` GPU bounded pilot、adaptive attacker 必须纳入，而且当前 blocker 不是“PIA 还能不能再调”，而是“PIA 之外的真实 second signal”和“更诚实的 defense successor”。       

你当前 admitted board 里，gray-box `PIA` 是 `AUC=0.841, TPR@1%FPR=0.059`；现有 `stochastic-dropout` 在 AUC 上只从 `0.841` 到 `0.828`，而你自己的 notes 已经明确 tail 比 average 更敏感。所以这轮最有价值的不是再追一个 headline AUC，而是找能改善 hard non-member tail 的第二信号，或者做一个真正 tail-aware 的 combiner / defense。  

先加一个统计学修正：**64/128 级别的 bounded pilot 几乎不可能稳定读出 `TPR@1%FPR`，`TPR@0.1%FPR` 更基本不能当 kill gate**。原因很直接：low-FPR 是极尾部统计，样本太小时它几乎退化成 one-false-positive-or-zero 的离散量。所以 `64` 只该做 smoke，`128` 只该做 calibration；真正的 low-FPR 判断至少要到 pooled `256/512` board，再报 bootstrap CI。你自己的 promotion 规则本来就要求 `256/512` 再 promotion，这一点我建议继续坚持。  

## 1) 五条优先级排序

**1. SimA single-query score norm**
这是当前最干净的 distinct family：你 repo 里 `PIA` 已经被明确写成 `epsilon-trajectory consistency`，而 SimA 把判别信号换成单时间步的 score / epsilon magnitude，即 (A(x,t)=|\hat\epsilon_\theta(x,t)|_p)。在 SimA 论文的 DDPM benchmark 里，它在 CIFAR-10 和 CIFAR-100 上都用 **1 query** 拿到了比 PIA 更高的 `TPR@1%FPR`；作者对 time-dependent attacks 做了 (t=0{:}300) sweep，SimA 用 (l_4) norm；官方仓库还直接给了 `DDPM/CIFAR10` attack script，所以实现摩擦最低。你本地 board 未必复现论文数字，但它是当前最像“真实 second signal”的候选。  ([arXiv][1])

**2. Tail-calibrated PIA + SimA fusion**
这不是 distinct family，但它是**最快**给 `05-cross-box` 和 low-FPR 交付价值的东西。ICML 2024 的 diffusion quantile-regression 工作已经在 diffusion MIA 里明确用了“每样本自适应阈值 / hypothesis test”，而更广义的 MIA 评估工作则显示：不同攻击会覆盖不同暴露成员，单一“最强攻击”往往低估 tail risk。所以在你这里，fusion 应被当成**honest calibration layer**，不是 leaderboard trick。 ([Proceedings of Machine Learning Research][2])

**3. Suspicion-gated perturbation defense**
作为 `all-steps dropout` 的 successor hypothesis，它比“再来一个 uniform randomness 变体”更值得做。AMG 明确做了 automatic detection + selective application，并且实验对象包含 DDPM 和 SD；ICLR 2024 的 memorization paper 用 prediction magnitude 做 early detection；attraction-basin 那条线则强调“只在某个 transition 之后/之前做 selective intervention”。共同方向都不是 blanket perturbation，而是**selective intervention**。但要强调：最强证据里的两篇是 text-conditional，所以对 unconditional `DDPM/CIFAR10` 来说，这是 principled translation，不是 paper-faithful reproduction。 ([arXiv][3])

**4. Mid-frequency residual / frequency-aware score**
这条线是“真有价值，但直达你当前 surface 的证据不如 SimA/Fusion 直接”。2025 的 frequency paper 证明了 plug-in 高频过滤能提升现有攻击，在 `STL10-U / CIFAR-100 / Tiny-IN` 和 text-to-image 上都有效；但我没有在那篇论文里看到直接的 `DDPM/CIFAR10` standalone scorer 结果。FCRE 则给了一个真正的 mid-band reconstruction formulation，但它是 medical diffusion。我的判断是：frequency 值得跑 bounded pilot，但不该排在 SimA 或 fusion 前面。 ([arXiv][4])

**5. Structured leakage-channel suppression**
这个方向有意思，但不该进当前 pure gray-box mainline。NeMo、Bright Ending、以及 memorized-subspace 这一支都支持“泄露可能局部化到少数 neurons / patches / subspaces”，但这些 strongest papers 基本都在 text-to-image、cross-attention、或 owner-side model surgery 上。放到 unconditional `DDPM/CIFAR10`，你需要 activations / channels / masks 的 owner-side instrumentation；按你自己的 threat model，这已经不是当前这轮 pure gray-box 该讲的东西了。 ([OpenReview][5]) 

## 2) 立刻实现的 2 条

**第一条：SimA**
因为它是最像“真实 second signal”的 distinct family，paper support 最直接，代码也最好接。 ([arXiv][1])

**第二条：PIA+SimA fusion**
因为它是最快给 `05-cross-box` 和 `04-defense` 提供可复用 tail layer 的东西，而且几乎不需要额外 GPU。 ([Proceedings of Machine Learning Research][2])

我**不会**把第一个 slot 用在 mid-frequency 上，因为 direct transfer 证据弱一档；也**不会**先上 gated defense，因为它最自然的 detector 就是 SimA。也不建议把 CLiD 拉进这轮：它是 text-conditioned 方法，SimA 论文自己的主 benchmark 也把它排除在 unconditional-heavy setup 之外。 ([神经信息处理系统会议论文集][6])

## 3) SimA、mid-frequency residual、PIA+SimA fusion 的具体实现规格

**SimA**
用 paper-faithful 版本：(s_S(x;t)=|\hat\epsilon_\theta(x,t)|_4)，分数越小越像 member。时间步不要盯着 (t=0)；论文明确说最早步不稳定、最佳信号通常出现在“early but not initial”的区间，所以 bounded scout 先扫一个小网格，例如 `t in {10, 20, 30, 50, 80, 120}`，若有信号再扩到官方风格的 step-10 sweep over `0:300`。流程上我会做：`64` 只 smoke 路径和符号，`128` 只锁 `t*` 和分数方向，`256` 才第一次看 pooled `TPR@1%FPR`，`512` 再决定是否 promotion。输出工件必须包含 raw score dump、`t*`、bootstrap CI、以及和 PIA 的 rank-correlation / hard-negative overlap。Kill gate 很简单：如果到 `256` 还既没有 standalone signal、又不能给 fusion 带来 tail lift，就把它从“headline second signal”降级成“fusion-only probe”。 ([arXiv][1])

**Mid-frequency residual / frequency-aware score**
这里必须分清 **A 版** 和 **B 版**。A 版是 `+F enhancer`：把 2025 频域 paper 的 filter 套在现有 residual scorer 上，比如 `PIA+F` 或 `SecMI+F`；它便宜，但不是新 family。B 版才是真候选：先做 one-step 或 short-5-step 的 (\hat x_0) reconstruction，再令 (r=x_0-\hat x_0)，对 (r) 做 channelwise DCT/FFT，只保留 annular mid-band，然后用 band-pass (L_2) 或 band-pass SSIM 当 membership score。band 只允许在 calibration split 上试 3 个固定候选，不许在 eval split 上扫。Kill gate 也要诚实：如果 B 版只有在“套在 PIA 上”时有效，而 standalone 不成立，就把它重命名成 enhancer，别把它包装成 second signal。因为 high-frequency 论文本身就是 plug-in wrapper，而 FCRE 才是 mid-band reconstruction method。 ([arXiv][4])

**PIA + SimA fusion**
先上**参数无关**、tail-oriented 的版本。做法是：只在 calibration non-members 上拟合每个分数的 empirical null CDF ( \hat F_i^{NM})；把各分数统一到同向后，转成 tail score，例如 (z_i(x)=\Phi^{-1}(\hat F_i^{NM}(s_i(x))))，然后用
[
S_{\text{fuse}}(x)=\max(z_{\text{PIA}}, z_{\text{SimA}})
]
做第一版组合。这个规则单调、tail-friendly，而且避免在小 calibration split 上学权重。只有当这个简单 combiner 已经有 low-FPR lift 时，才允许再试 learned isotonic 层。frequency 只有在 standalone 成立，或者至少明显改变了 hard-negative set，而不只是给 PIA 排序去噪时，才进入 tri-signal 版本。这个接口以后还能直接接 `Recon` 或 `GSA` 的 calibrated score，所以它是最自然的 `05-cross-box` 过渡层。 ([Proceedings of Machine Learning Research][2])

## 4) 是否值得把 suspicion-gated defense 作为当前 all-steps dropout 的 successor

**值得做，但只值得当 bounded successor hypothesis，不值得立刻当 admitted replacement。**
具体做法：用最佳 SimA probe timestep 作为 suspicion score，只把 top-`q`（如 `5/10/20%`）的可疑样本送入 `late-step dropout` 或轻量 late-step epsilon perturbation，其余样本走 clean path。评估必须 defense-aware：做 `3/5` 次 repeated-query aggregation，固定总 query budget，与当前 `all-steps dropout` 做 matched-cost、matched-quality 比较，并报四个指标的均值/方差。Kill gate 也应很硬：如果 adaptive attacker 轻易把 tail 恢复回来，或者在相同质量/成本下并不优于当前 dropout，就尽快停。它值得做，是因为 2024–2025 的 mitigation 主流越来越 selective；它暂时不能升格，是因为 strongest paper support 仍然偏 conditional。  ([arXiv][3])

## 5) 哪些是真正 distinct family，哪些只是 calibration / enhancer

**真正 distinct family**

* `SimA`: 是。它是 single-step score-magnitude / local-geometry family，不是 trajectory-consistency。 ([arXiv][1])
* `Mid-frequency residual`: **只有在 standalone band-pass residual 成立时才算是**；如果只是 `PIA+F`，那不是新 family。 ([arXiv][4])
* `Suspicion-gated perturbation defense`: 是新的 defense family。 ([arXiv][3])

**不是 distinct family**

* `PIA+SimA fusion`: 不是，是 calibration / ensemble layer。 ([Proceedings of Machine Learning Research][2])
* `Frequency filter on top of PIA/SecMI`: 不是，是 enhancer。 ([arXiv][4])

**gray-box-plus / owner-side**

* `Structured leakage-channel suppression`: 该这么标，而不是硬塞进当前 pure gray-box。 ([OpenReview][5]) 

## 6) 每条的 minimal bounded pilot、成本估算、kill gate

这里我按你当前 `RTX 4070 8GB + CPU-first + <=4h GPU pilot` 的约束给 rough order，而不是伪精确时长。  

**SimA**
最小 pilot：`64` smoke → `128` calibration → `256` first tail read → `512` full board。
成本：在 fixed timestep 下是 **1 query / sample**，paper 和 code 都比 PIA 轻；真正耗时主要来自 timestep sweep，不是单点评分。
kill gate：`256` 上既无 standalone signal，也不给 fusion 带来 low-FPR lift，就停。 ([arXiv][1])

**Mid-frequency residual**
最小 pilot：`64` 只验 residual path；`128` 锁 band 与 timestep；`256` 才看是否真的改变 tail。
成本：A 版 enhancer 几乎是 residual dump 之后的 CPU-side FFT；B 版 standalone one-step 版本是 GPU-light，short-5-step 会靠近 PIA 级别。
kill gate：如果只在 `+F on PIA` 时有效，就降级成 enhancer；如果 standalone 在 `256` 上没有任何 tail 改善，就停。

**Fusion**
最小 pilot：不用新 GPU run，直接在 `PIA` 与 `SimA` score dump 上做 `128` calibration → `256` validation → `512` full。
成本：几乎 CPU-only。
kill gate：如果不能在 `256` 上超过 best single score 的 low-FPR tail，就停；AUC 涨而 tail 不涨，不算过关。 ([Proceedings of Machine Learning Research][2])

**Suspicion-gated defense**
最小 pilot：`64` 只验 gating plumbing；`128` 锁 `q` 和 late-step window；`256` 做 defense-aware repeated-query eval。
成本：大致是“当前 defense + 一个便宜 probe”的量级；只要 `q` 稀疏，仍应能压在单 slot 内。
kill gate：matched quality/cost 下若不优于 current `all-steps dropout`，或 adaptive `3/5-repeat` 攻击把抑制效果拿回来，就停。 

**Structured suppression**
最小 pilot：`128` owner-side activation logging + 小 mask ablation。
成本：工程最贵。
kill gate：只要当前 lane 不接受 owner-side instrumentation，直接不进 pure gray-box backlog。

额外补一句：**`TPR@0.1%FPR` 在 bounded pilot 阶段只该记录，不该当 hard gate**。在你这组 packet size 上，它几乎永远是极度离散且高方差的。

## 7) 如果目标是最快给 05-cross-box 和 04-defense 提供可复用第二信号，先做什么

先做 **SimA**。
紧接着做 **PIA+SimA fusion**。
然后把 **同一个 SimA probe** 复用到 suspicion-gated defense。
`mid-frequency residual` 放到第三顺位，只在前两条已经稳定时再上。这样一个信号能同时服务三件事：给 gray-box 一条真实 second signal，给 `05-cross-box` 一个可扩展的 calibrated score layer，给 `04-defense` 一个 sample-aware detector。你的 `I-C` 本来就是 fusion 方向，所以这条顺序最短。  

最后给一句最短执行版：**一个 `<=4h` GPU slot，我会用在 `SimA scout -> score dump`；同一轮 CPU sidecar 直接做 `PIA+SimA fusion`；只有这两步正了，下一轮才给 suspicion-gated defense，而不是先去追 mid-frequency。**

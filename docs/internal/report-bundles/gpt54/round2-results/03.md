## H1: Activation-subspace fingerprint attack

### 1.1 Layer and timestep selection

Use **Option C**, not A. The right adaptation is to keep the **same projection type across nearby U-Net depths**, not to switch to `to_k` or `to_out` inside the same block. NeMo’s strongest localization signal in text-conditioned diffusion was concentrated in **value mappings**, especially early/down-block cross-attention value layers, and their method relies on activation differences relative to holdout activation statistics. For your unconditional DDPM, the clean analogue is three self-attention value projections across depth, for example: nearest available `down_blocks[*].attentions[*].to_v`, `mid_block.attentions.0.to_v`, and nearest available `up_blocks[*].attentions[*].to_v`. ([NIPS 论文集][1])

For timesteps, do **not** use uniform full-schedule spacing like 250/500/750 as the first scout. Use a **front-loaded high-noise schedule** because diffusion memorization signals are repeatedly reported as strong at the first denoising step or very early in the reverse process, and NeMo itself computes its core noise-difference signal from the **first denoising step**. My default 3-step scout is `t = {999, 749, 499}` under the standard DDPM convention where larger `t` is noisier. If your timestep convention is reversed, just map these to reverse-time early/mid steps. ([NIPS 论文集][1])

For scaling after a positive 64-sample scout, add **more layers first**, not more timesteps. Extra hooked layers at the same timestep are almost free relative to extra forward passes, while more timesteps multiply runtime. So the expansion path should be: `3 layers × 3 steps` → `5 layers × 3 steps` → only then `5 layers × 5 steps`.

### 1.2 Per-channel statistics

Use four core statistics per channel per timestep:

`mean_abs = mean(|A_c|)` over spatial positions
`var = var(A_c)` over spatial positions
`q95_abs = quantile(|A_c|, 0.95)` over spatial positions
`ood_frac = mean(|(A_c - μ_hold)/(σ_hold + 1e-6)| > 2)`

The fourth term is the one I would use instead of a literal zero-threshold sparsity metric, because attention/value outputs in DDPM are not naturally sparse in the ReLU sense, while NeMo is explicitly built around **outlier activation detection** using holdout mean and standard deviation. That makes `ood_frac` more stable and more literature-aligned than “fraction below ε.” ([NIPS 论文集][1])

Do **not** do PCA per channel. Use a **shared PCA/SVD sketch over the full standardized feature matrix** after the four hand-crafted stats are computed. For the scout, keep `k=8` global PCs. That captures cross-channel co-activation structure, which is the actual “subspace” part of H1. ([arXiv][2])

Across timesteps, compute stats **per timestep and concatenate**. Do not pool across time first. Add only two temporal deltas as cheap dynamics features: `Δ mean_abs` and `Δ ood_frac` between adjacent chosen timesteps. That gives you time sensitivity without exploding feature count.

### 1.3 What “top 32–128 features” should mean

Use a **channel-first, feature-second** pipeline.

First, rank channels on shadow data using **random-forest feature importance** over each channel’s per-timestep stat vector. The strongest evidence I found for hidden-activation selection says random forest was the best-performing neuron-selection strategy, with the informative mass concentrated in a relatively small subset of neurons. ([arXiv][3])

Then freeze the top channels and feed their individual stat/time features into the attack model. So “top 32–128 features” should mean **top individual stat/time features, but only from the shadow-selected channels**. For the first scout, target **8 channels total** across the three layers. With 4 stats × 3 timesteps, that gives 96 features before PCA; if you add the 8 global PCs, you land at 104, which is right in your desired band.

For ranking stability, aggregate channel ranks across the 3 shadows and only keep channels that appear in the top set for **at least 2 of 3 shadows**. That is the simplest useful stability-selection rule.

### 1.4 Shadow calibration

Use **3 shadow models**, same architecture and hyperparameters as the target, different random seeds only. Do **not** vary hyperparameters for calibration. Your repo already treats 3 shadows as the balanced default for white-box comparison.  

Use disjoint member subsets per shadow. Controls should also be disjoint if convenient, but because CIFAR10 has abundant held-out data, overlap is unnecessary. Feature selection must be done on **shadow data only**, then frozen and applied unchanged to the target.

Train a single pooled **L2 logistic regressor** first. Only try a 1-hidden-layer MLP if LR clearly underfits. Calibrate two outputs from shadows: a continuous score for ROC/AUC, and a threshold chosen on shadow validation to hit **1% FPR**. That keeps the attack aligned with your low-FPR-first reporting standard.  

## H2: Score-vector geometry attack

### 2.1 Timestep selection

The official SimA DDPM code computes score features across multiple fixed slices, and the released DDPM scripts use `attack_num=30` and `interval=20` for CIFAR10/CIFAR100, while the attack itself is explicitly framed as a score-based, efficient alternative centered on the denoiser output. ([GitHub][4])

For your bounded scout, keep timesteps **fixed across all samples** and **shadow-tuned**, not adaptive per sample. My default 4-step scout is `t = {999, 799, 599, 399}`. If it survives 128, promote to an 8-step grid `{999, 899, 799, 699, 599, 499, 399, 299}`. That keeps the grid concentrated where early reverse-time signals are most likely to be stable, without paying for a full 30-step SimA-style sweep.

### 2.2 Norm and angle features

The most important concrete detail from the official SimA DDPM implementation is that its score uses an **L4-type energy**, specifically `sum(|eps_hat|^4)` over spatial dimensions, not plain L2. So your primary per-timestep feature should be:

`sima_t = sum(|eps_hat_t|^4)`

The official SimA code literally defines the distance as `(x0.abs()**4).flatten(2).sum(dim=-1)` on the collected epsilon estimates. ([GitHub][5])

Add two secondary features per timestep:

`norm2_t = ||eps_hat_t||_2 / (||x_t||_2 + 1e-6)`
`cos_t = cos(eps_hat_t, eps_true_t)`

Use the cosine with the **true injected Gaussian noise** used to form `x_t`, not cosine across different timesteps. The former is physically meaningful; the latter mostly tracks schedule geometry.

Aggregate by concatenation, then append three global summaries computed across timesteps: `mean(sima_t)`, `std(sima_t)`, and slope from a linear fit over timestep index. A tiny LR is enough; no MLP needed here.

### 2.3 Low-pass filtering

Use the frequency paper’s **Fourier-domain mask**, not Gaussian blur, and apply it to a **reconstruction-domain residual**, not to the denoiser input. Their method explicitly defines a frequency mask `β(r)` that attenuates frequencies above a cutoff radius `r_t`, and they show CIFAR-scale datasets using small cutoff radii, with CIFAR-100/Tiny-IN analysis using radius 2 as the high/low boundary. ([arXiv][6])

For DDPM/CIFAR10, my default auxiliary feature is:

`lp_err_t = || LPF(x0_pred_t - x0) ||_2`

where `x0_pred_t` is recovered from `x_t` and `eps_hat_t`. Start with a Fourier mask using `r_t = 2` and attenuation factor `s = 0.2`. That exact `s` is an adaptation rather than a paper-mandated CIFAR10 default, but it sits in the robust part of their sweep and is a sensible scout value. Then grid-search `s ∈ {0.0, 0.1, 0.2, 0.3}` and `r_t ∈ {1, 2, 3}` on shadows only. ([arXiv][6])

## H4: Risky-subspace pruning defense

### 4.1 Shared subspace identification

Use **H1 activations only** for the first H4 packet. Do not broaden the activation source until H1 itself proves stable. The right extraction is a **discriminative SVD**, not plain PCA on stacked members and controls. Concretely:

1. z-score H1 features using control `μ, σ`
2. subtract the control mean
3. stack member residual vectors into a matrix `M`
4. compute SVD on `M`
5. take top `k ∈ {1,2,4,8}` right singular vectors as the risky directions

This is closer to the “memorized prompts activate a common subspace” story than unsupervised PCA on all samples, which would mostly capture nuisance variance. ([arXiv][2])

Choose `k` by **smallest effective k**, not explained variance. I would grid `{1,2,4,8}` and pick the smallest value that reduces H1 score materially without a visible quality collapse.

### 4.2 Zeroing vs projection vs pruning

For a reversible scout, do **activation-space channel zeroing** first. Zero activations, not weights. Start with the top **4 channels total** or about **1–2% of channels in the targeted layers**, whichever is smaller.

For the real H4 mainline, use **orthogonal activation projection**:

`A' = A - U_k(U_k^T A)`

applied at the selected hooked layers. This is the cleanest implementation of “remove the risky subspace” and is more faithful to the subspace hypothesis than blunt channel zeroing.

For the permanent edit follow-up, use **weight pruning aligned to the risky subspace** inside the selected `to_v` / `to_out` matrices. Do not use global magnitude pruning, and do not base pruning on per-sample gradients for this first packet. The selection criterion should be weight alignment to the learned risky directions.

### 4.3 What 0.05%–0.5% means

Interpret that ratio as **percentage of weights in the targeted layers only**, not the entire model. Whole-model pruning at that range is too blunt for this hypothesis.

Grid `{0.05%, 0.1%, 0.2%, 0.5%}` over the targeted matrices. Pick the smallest ratio that yields a real attack reduction on H1 or H2 while keeping quality degradation under your pre-declared bound. Because your own repo already has a negative single-selector falsifier on Finding NeMo-style attenuation, I would treat **projection** as the real candidate and **pruning** as the permanentization step only if projection works.  ([arXiv][2])

After any edit, re-run **GSA, H1, and H2**, plus a lightweight quality packet. If you have time for only one utility metric, use FID/IS or your nearest existing generation-quality proxy on held-out data.

## H5: Targeted unlearning defense

### 5.1 Forget-set scoring and size

Use an **ensemble risk score**, not a single-family score. Concretely, rank-normalize and average:

* GSA score
* H1 activation-fingerprint score
* H2 score-geometry score

Then take the risky shortlist and diversify it. Pure top-k often over-concentrates on one visual motif or one class. For CIFAR10, I would shortlist the top `2K` risky members, cluster them in image or embedding space, and keep `K` medoids with loose class preservation.

Use:

* `K = 32` for the first pilot
* `K = 64` as the default serious pilot
* `K = 128` only if 64 works and the trainable parameter budget stays tiny

### 5.2 PEFT vs full fine-tuning

For your repo, the safest first packet is **selective unfreezing**, not broad full fine-tuning. Unfreeze only the attention value/output projections in the H1-localized neighborhood: `mid_block` plus the nearest up/down attention blocks.

If your codebase already has LoRA plumbing on this DDPM, then LoRA with `r = 4` or `8` on those same modules is a good drop-in replacement. If not, do not burn time adding LoRA infrastructure just for this packet.

Keep trainable parameters under about **2%** for the first H5 attempt.

### 5.3 Objective

Use a **retain + forget mixture**, not pure NegGrad. The strongest concrete guidance here is SISS: it combines naive deletion and NegGrad through importance sampling, cuts the forward-pass cost in half relative to the two-forward formulation, and the authors explicitly warn that NegGrad alone degrades retained quality quickly. Their README also recommends clipping the forget-side gradient to about **10% of the naive deletion gradient norm**. ([ar5iv][7])

So the preferred objective is a SISS-style mixture. If you want the simpler fallback, use:

`L = L_retain - λ L_forget`

with:

* `retain set` = random retained training samples, preferably low-risk ones
* `λ` start around `0.3`
* `lr` in the `3e-6` to `1e-5` range
* gradient clipping on the forget term
* early stopping on the privacy/quality Pareto frontier

For the first scout, budget **200–500 update steps**. Only go beyond that if the privacy signal is still moving and quality is not.

Verification should include:

* attack-score drop on the forget set under GSA/H1/H2
* held-out quality metric
* a check that the effect is not just overfitting to the forget subset

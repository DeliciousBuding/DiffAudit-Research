# E2SCT-012 Shake-to-Leak Public-Surface Check

> Date: 2026-06-06
> Scope: no-download public-surface check for the measurement-route gap board.

## Question

Does `E2SCT-012` expose a released private-set manifest, fixed checkpoint,
generated response packet, SecMI/output score array, ROC/metric JSON, or
verifier that can make it an E2 response/score evidence row?

## Checked Public Surface

No repository clone, archive, image payload, dataset, model, checkpoint, or
generated output was downloaded. The check used public metadata, GitHub pages,
README text, and small raw script files only.

## Finding

`E2SCT-012` is a clean code-availability false-promotion exemplar. The public
repository is mechanism-relevant and runnable, but the audited artifacts are
runtime products:

- synthetic private-set images are generated by `sp_gen.py`;
- fine-tuned targets are written to local `./ckpts/<domain>/`;
- SecMI scores require local `laion-2b`, `celeb_and_web`, and checkpoint paths;
- extraction candidates are generated locally from a fine-tuned model.

The allowed public surface does not expose:

- a frozen private-set manifest with immutable member/nonmember image ids;
- fixed target checkpoints with size/hash/training binding;
- generated response packets;
- per-sample SecMI/output score arrays;
- ROC arrays, metric JSON/CSV, or a no-training verifier.

| Metric provenance | `Fail` | No public ROC arrays, metric JSON/CSV, or verifier were found. |

| Consumer/delta | `Fail` | The current surface supports code-availability overpromotion analysis, not a consumer-ready audit result. |

## Decision

`code_availability_false_promotion_exemplar / no_frozen_packet /
no_compute_release`.

Do not count `E2SCT-012` as admitted evidence, a response/score asset, or an
external-audit denominator row. Keep it as a false-promotion exemplar candidate
for the measurement route: code availability and a strong paper claim are not
enough without public frozen private-set, checkpoint, response, score, and
metric artifacts.

Allowed wording:

`Shake-to-Leak is public runnable code for a fine-tuning-amplified generative
privacy mechanism, but the public surface does not expose frozen private-set
manifests, target checkpoints, generated response packets, score arrays, metric
JSON, or a verifier. It is a code-availability false-promotion exemplar, not
admitted response/score evidence.`

## Reopen Condition

Reopen only if the authors publish checkpoint-bound score artifacts or a compact
public packet that binds private-set image ids, generated outputs or SecMI score
rows, checkpoint identity, member/nonmember roles, and recomputable metrics. Do
not download Stable Diffusion weights, LAION/person images, synthetic private
sets, checkpoints, generated payloads, or model artifacts for this gate.

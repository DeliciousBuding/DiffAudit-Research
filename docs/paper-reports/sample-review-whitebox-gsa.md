# 扩散模型白盒成员推断攻击

**英文题目**：White-box Membership Inference Attacks against Diffusion Models

这篇论文讨论扩散模型在强白盒条件下的成员推断问题。作者关注的不是生成结果本身，而是模型对样本的内部响应是否暴露了成员信号。论文的核心判断是，单个时间步上的 loss 难以稳定表达这种差异，更有效的对象是跨时间步、跨层组织的梯度特征。围绕这一判断，文章提出 `GSA` 框架，并在 DDPM 与 Imagen 上给出接近饱和的攻击结果。

## 1. 论文定位与核心问题

这篇论文在 DiffAudit 文献体系中属于白盒路线的主论文。它研究的是高权限条件下的成员泄露上界，也就是在攻击者已经拿到模型参数、结构并可执行样本级反向传播时，训练成员与非成员是否还会表现出稳定且可学习的差异。相比黑盒和灰盒方法，它不直接服务于线上 API 风险判断，但非常适合作为白盒路线的强参考点。

论文要解决的核心问题有两个。第一，扩散模型白盒成员推断究竟该用什么特征。第二，如果梯度比 loss 更有信息量，那么怎样把高维梯度压缩成可训练的攻击表示。作者的回答很明确：成员信号应从多时间步梯度中提取，再通过分步采样和层级聚合形成低维特征。

## 2. 方法概览

`GSA` 的基本流程是：先从总扩散步数中选择一个时间步集合 `K`，然后在每个采样时间步上构造带噪样本、计算 loss、执行反向传播，并提取样本级梯度。由于原始梯度维度很高，论文不直接保留参数张量，而是把每层梯度压缩为范数统计量，再在时间步维度上做平均聚合，最后将聚合后的表示送入攻击分类器。

作者给出了两个实例化方法。`GSA1` 先对多个时间步的 loss 求均值，再进行一次反向传播，因此计算代价较低。`GSA2` 则对每个时间步分别提取梯度，再对多时间步梯度做平均，因此保留更多信息，但运行时间更长。两者共同说明，白盒成员推断的关键不在分类头设计，而在梯度信号如何被组织。

## 3. 关键公式与技术含义

扩散前向过程定义了模型在第 `t` 个时间步所看到的带噪样本：

$$<equation>x_t = \sqrt{\bar{\alpha}_t}x_0 + \sqrt{1-\bar{\alpha}_t}\,\epsilon_t</equation>$$

这条公式决定了成员信号的观察空间。论文强调，不同时间步对应不同噪声状态，因此攻击者观察到的不是原图上的统一响应，而是样本在整条扩散轨迹中的响应差异。

更关键的是梯度表达式：

$$<equation>\nabla_\theta L_t(\theta,x)=2\left(\epsilon_\theta(x_t,t)-\epsilon_t\right)^\top \nabla_\theta \epsilon_\theta(x_t,t)</equation>$$

这里同时包含预测误差项与模型局部响应项。loss 只保留前者的标量结果，梯度则把误差与参数敏感性一起编码。作者据此认为，成员与非成员即便具有接近的 loss，也可能在局部响应结构上显著不同，因此梯度更适合作为白盒攻击特征。

在具体实现上，论文把每层梯度压缩为 `\ell_2` 范数，并在时间步上做平均聚合。`GSA1` 倾向于降低成本，`GSA2` 倾向于保留更多时间步信息。这个设计使方法既保留了层级结构，又避免了直接处理超高维梯度张量。

## 4. 实验设置与主要结果

实验覆盖无条件扩散模型和条件扩散模型两类场景。无条件部分使用 DDPM，并在 CIFAR-10 与 ImageNet 上训练；条件部分使用 Imagen，并在 MS COCO 上训练。指标采用 `ASR`、`AUC`、`TPR@1%FPR` 和 `TPR@0.1%FPR`。基线除 threshold-based loss 外，还包含 LiRA、Strong LiRA，以及在相同采样条件下只使用 loss 特征的 `LSA*`。

主结果首先体现在梯度特征相对 loss 特征的明显优势。在 CIFAR-10 上，`GSA1` 与 `GSA2` 的 `AUC` 都达到 `0.999`，而 `LSA*` 只有 `0.909`。在低误报区间，`GSA1` 的 `TPR@1%FPR` 达到 `99.7%`，说明该方法不只是整体可分性强，在高置信判断下也保持了很高的识别力。

## 5. 局限与复现判断

论文的主要局限来自威胁模型。它要求攻击者能够访问参数并提取样本级梯度，这一条件在多数闭源服务中并不成立。因此，文中的强结果更适合作为白盒上界，而非黑盒风险陈述。与此同时，梯度提取的计算代价也明显高于直接计算 loss，模型规模增大后实验预算会上升很快。

若要在 DiffAudit 中做相对忠实的复现，至少需要目标模型 checkpoint、成员与非成员划分、样本级梯度提取接口、影子模型训练管线和攻击分类器特征导出流程。当前真正的阻塞不在论文理解，而在高权限实验链尚未打通。比较稳妥的落地顺序，是先在小规模 DDPM 上完成梯度提取和时间步采样原型，再决定是否扩展到更大模型。

## 6. 对 DiffAudit 的直接价值

这篇论文对 DiffAudit 的第一价值，是为白盒路线提供了能力上界。它清楚说明，一旦能够获取参数与梯度接口，成员信号可以非常强。第二价值是实现优先级上的指导：white-box 原型最需要补齐的是样本级梯度提取、时间步采样和层级聚合，而不是继续堆叠新的打分函数。第三价值是叙事边界的澄清，它提醒团队不要把白盒结果直接外推到黑盒场景，而应把它作为高权限条件下的强证据。

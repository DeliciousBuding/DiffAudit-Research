## Gate Result

| Gate | Result |
| --- | --- |
| Target model identity | Fail for replay. DDIM/DCGAN training commands and configs are public, but no frozen trained target checkpoint with size/hash/training binding is released. |
| Exact member split | Partial. The runtime code deterministically constructs train/member subsets from dataset indices and seed values, but no immutable committed split manifest is shipped. |
| Exact nonmember split | Partial for runtime reconstruction, fail for public replay. Nonmembers are generated by the same runtime seed-derived indexing, but no committed nonmember manifest is available. |
| Query/response or score coverage | Fail. The repo ships no generated sample packet, `pos_result.npy`, `neg_result.npy`, ROC arrays, metric JSON, or ready verifier output. |
| Mechanism delta | Pass as a direct diffusion-MIA code reference. It includes white-box DDIM loss and black-box nearest-generated-sample attack code, not another CommonCanvas/MIDST/Beans/Fashion-MNIST variant. |
| Current DiffAudit fit | Watch-plus / code-public. The artifact is stronger than paper-only because official code and FID stats are public, but it would still require dataset acquisition, target training/sampling, and local attack execution from scratch. |
| Download justification | Fail. Downloading CIFAR-10/CelebA or training DDIM/DCGAN targets would start a reproduction project without recovering public checkpoint-bound score artifacts. |
| GPU release | Fail. There is no frozen target, reusable score packet, metric contract, or bounded stop condition that justifies GPU execution in this roadmap cycle. |

## Decision

`official-code-public / runtime-split-construction /
checkpoint-and-score-artifacts-missing / no download / no GPU release / no
admitted row`.

`fseclab-osaka/mia-diffusion` should be retained as an official direct
diffusion-MIA code reference and watch-plus item. It is useful evidence that
the older DDIM/DCGAN MIA line has public attack code, deterministic runtime
split construction, and FID-stat caches, but it is not a replay target because
it does not ship the checkpoint-bound member/nonmember scores, ROC arrays,
metric packets, generated samples, target checkpoints, or immutable split
manifests needed by DiffAudit.

Smallest valid reopen condition:

- Public DDIM or DCGAN target checkpoint bundles with size/hash and training
  seed/data binding;
- Matching immutable member and nonmember split manifests for the exact
  checkpoint;
- Generated sample packets or raw member/nonmember scores plus ROC arrays or
  metric JSON; and
- A bounded verifier command that reads public artifacts without training
  targets, downloading large datasets, or regenerating response packets.

Stop condition:

- Do not clone the full repository just to run the scripts.
- Do not download CIFAR-10, CelebA, DDIM/DCGAN checkpoints, generated samples,
  or any model/dataset payload from this gate.
- Do not run `ddim/main.py`, `dcgan/train.py`, `dcgan/generate.py`,
  `attacks/fa_ddim.py`, `attacks/fa_dcgan.py`, `attacks/fbb.py`, or TTUR FID
  evaluation in the current roadmap cycle.
- Do not promote FSECLab MIA-Diffusion into Platform/Runtime rows, product
  copy, recommendation logic, or the admitted evidence bundle.

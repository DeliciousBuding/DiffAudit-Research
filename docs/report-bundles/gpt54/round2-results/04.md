# Round 2 defense implementation brief

先给执行结论：

* **Do now #1:** **H1 Risk-targeted SISS unlearning**
* **Do now #2:** **H2 Privacy-aware adapter fine-tuning / MP/SMP-DDPM**
* **Cheap sidecar only:** **H3 Sensitivity-conditioned noise**
* **Heavy fallback only:** **H4 DistillMD-lite with bottleneck**
* **Not now:** **H5 Stronger latent bottleneck / compression defense**

这个排序的依据很窄也很明确：你现在要解的不是“哪条线长期最有论文味”，而是“在 admitted `DDPM/CIFAR10` 表面上、`<=4h` RTX 4070 级 pilot、以 `TPR@1%FPR / TPR@0.1%FPR` 和 adaptive review 为中心，哪条线最可能给出 honest gain”。现有 repo 里 `stochastic-dropout` 已经是便宜但不够狠的 defended comparator，`DPDM / W-1` 又太重；因此新候选必须比前者更强、比后者更轻。     

把 **H1** 放第一，是因为它同时满足三点：**post-training**、**直打 high-risk member tail**、**天然兼容你现有的 PIA/GSA 资产**。SISS 本身就是 diffusion unlearning 里相对直接、成本受控的一条线，而且它把 naive deletion 与 negative-gradient unlearning 统一到 importance-sampled formulation 里；但 diffusion unlearning 近作也明确提醒，很多“forgetting”最后只是 concealment，所以它必须配套 subset audit 和 defense-aware attacker。 ([arXiv][1])

把 **H2** 放第二，是因为它是当前最像“比 dropout 更强、比 DPDM 更轻”的训练时 shaping 路线。MP-LoRA / SMP-LoRA 直接把 proxy MIA signal 写进轻量 adapter 微调目标里，论文里对 `λ` 和 rank 都做过消融，`λ≈0.05` 是一个不错的 privacy-quality 起点，而且 rank 在较大范围内不算特别敏感；这非常适合你现在的 bounded pilot 思路。 ([arXiv][2])

把 **H3** 降到 sidecar，不是因为它没价值，而是因为它最容易被 **repeated-query aggregation** 重新打穿；而且更“高级”的 higher-order Langevin 变体虽然有想法，但当前公开证据主要在 toy/speech 侧，**没有**给出对你这条 `DDPM/CIFAR10` 主面的 decision-grade overhead 读数。它适合作为便宜 scout，不适合作为本轮 headline。 ([arXiv][3])

**H4** 和 **H5** 都不该进当前主槽位。DualMD / DistillMD 说明“split + distill”在 diffusion defense 里是有路数的，但更一般的蒸馏文献也显示 student 可能比 teacher **更**泄露；而 latent bottleneck 虽然从 SimA 角度看是合理长期方向，但它本质上已经在改架构，不是当前 admitted 表面的 bounded successor。 ([arXiv][4])

## 1) Ranked recommendation

**立即做**

1. **H1 Risk-targeted SISS unlearning**
2. **H2 Privacy-aware adapter fine-tuning / MP→SMP-DDPM**

**保留为 fallback / sidecar**
3. **H3 Sensitivity-conditioned noise**
4. **H4 DistillMD-lite with bottleneck**

**当前不要做**
5. **H5 Stronger latent bottleneck / compression defense**

额外判断：**现在不要做组合防御**。先让单条线在 `128` packet 上、adaptive review 下，真实超过 `stochastic-dropout` 的低 FPR 表现，再谈 `H1+H3` 或 `H2+H3`。不然只会把搜索空间做大，而不是把证据做硬。  

## 2) Implementation spec for the top 2

### H1: Risk-targeted SISS unlearning

**直接答案**

* **top-k ranking signal：**不用只看 `GSA`，也不用只看 `PIA`。先把两者都变成 **within-member percentile rank**，默认
  `risk(i) = 0.5 * q_GSA(i) + 0.5 * q_PIA(i)`。
  先检查 `Top10%(GSA) ∩ Top10%(PIA)`：如果交集足够大，优先从交集里选；否则用平均 percentile 排序。
* **`k ∈ {16, 32, 64}`：**合理。默认 **`k=32`**；`16` 做 cheapest scout，`64` 做 stress rung。
* **objective：**不要纯 forget loss，也不要纯 NegGrad。用 **retain+forget hybrid**。
  最实用的第一版是：

  * 以概率 `1-λ`：只做 `L_keep`
  * 以概率 `λ`：做 `L_keep - α * L_forget`
    这是一个对 SISS “deletion + NegGrad mixture” 的 repo-friendly 落地。
* **默认超参数：**`λ=0.5`，`α=0.5`，retain:forget batch 比例 `3:1`，`lr=1e-5`，`grad_clip=1.0`，steps 用 `1000 / 2000 / 4000` 对应 `64 / 128 / 256` packet。
* **最小资产：**admitted checkpoint、现有 member 风险分数导出、forget-set id list、当前 PIA/GSA evaluator。

**文献锚点**

SISS 的关键点不是“暴力反向优化 forget set”，而是把 naive deletion 和 NegGrad 统一进 importance-sampled unlearning；这正是为什么我建议你实现成 retain+forget hybrid，而不是单一 forget objective。最直接的 paper anchor 是 *Data Unlearning in Diffusion Models* 的 **Eq. (5)–(8)**。同时，diffusion unlearning 近作明确提示：如果只看表面 attack score 下降，可能把 concealment 误判成 forgetting。([arXiv][1])

**fallback heuristic**

如果第一轮不想把论文式 importance sampler 全部接进来，就先上这个最小实现：

```text
for each step:
  sample B_keep
  with prob 0.5:
      optimize L_keep
  else:
      sample B_forget
      optimize L_keep - 0.5 * L_forget
```

这已经足够回答“这条线值不值得继续”，而不需要先把 unlearning 工程做成完整子项目。这个策略也符合你当前 `CPU-first / bounded pilot / no waste` 的治理约束。 

**敏感度**

* **高：**forget-set ranking、`k`、`α/λ`
* **中：**steps
* **低：**是否先做 `16` 再到 `32`

### H2: Privacy-aware adapter fine-tuning / MP→SMP-DDPM

**直接答案**

* **adapter 插位：**默认只插在 **U-Net middle block + 最后两个 up blocks**。

  * 如果当前 DDPM 有 attention：插 `q / v / out` 投影。
  * 如果 attention 很少或没有：改成同一位置上的 `1x1 conv / linear output projection` 低秩 adapter。
* **proxy MIA penalty：**第一版别上来就做复杂 ratio objective。先做 **MP-style min-max**：

  * 训练一个小攻击头 `h_φ(f_θ(x))` 区分 member / non-member
  * adapter 更新时最小化 `L_ddpm - λ * L_attack`
    如果这条线在 `64/128` 上有信号，再升到 **SMP-style** 比值形式。
* **feature cache：**`64/128` scout 先 **frozen cache**；只有 promotion 到 `256` 以后再加 periodic refresh，每 `1000` step 刷一次。
* **默认超参数：**

  * `rank ∈ {4, 8}`，默认先跑 `r=4`
  * LoRA scale `≈ 2r`
  * `lr = 5e-5` 起步，可上探 `1e-4`
  * `λ ∈ {0.02, 0.05, 0.1}`，默认 **`0.05`**
  * warmup `500` step 只训 `L_ddpm`
  * 之后总 steps `3000–6000`
* **merge adapter 后的 inference：**可以近似按 `1x` 看，不该成为当前 pilot 的主成本项。

**文献锚点**

MP-LoRA / SMP-LoRA 的核心是把 proxy MI signal 写进 adapter 目标，而不是靠全模型 DP 训练来压泄露。论文里 LoRA 是挂在 backbone attention 层上的；`λ` 消融显示 `0.05` 左右是不错的 privacy-quality 起点，rank 消融则表明 rank 不是最敏感维度。你这里从 LDM attention-only 迁到 DDPM middle / late-up block，是**工程适配**，不是 paper-faithful 复刻；但方向是对的。([arXiv][2])

**fallback heuristic**

如果攻击头太不稳，先不用完整 `h_φ`，改成更便宜的 proxy：

* 取 4 个最可分 timestep 的 `||ε_θ(x_t,t)||_2`
* 用一个 logistic head 做 member / non-member
* 先把这条线跑通，再升级为全 PIA-style feature

这一步的目的不是“证明 SMP-DDPM 已完整复现”，而是验证：**轻量 objective shaping 能否在 low-FPR 上超过 dropout 基线**。 

**敏感度**

* **高：**layer placement、`λ`、proxy feature 选取
* **中：**rank、cache refresh 周期
* **低：**merge 与否

### H3: Sensitivity-conditioned noise（只做 cheap sidecar）

**直接答案**

* **高风险 timestep 定义：**从当前 `PIA` traces 上，算每个 timestep 的 member / non-member separability；用 `AUC_t` 或 Cohen’s `d_t` 排序。先取 **top-4**（64 packet）或 **top-8**（128 packet）。
* **fixed vs entropy-conditioned：****先做 fixed noise**。因为 unconditional DDPM 上“entropy-conditioned”本身就缺一个天然、便宜、稳定的 uncertainty proxy。
* **噪声插位：**第一版只插到 **`ε` prediction**，不要先动 activation，更不要先上 sampler dynamics。
  推荐形式：`ε̃_t = ε_t + σ_t z`，`z ~ N(0, I)`，只在选中的高风险 `t` 上加；`σ_t = c * std(ε_t)`，`c ∈ {0.01, 0.02, 0.05}`。
* **higher-order Langevin 不进当前 pilot：**这里的 auxiliary variables 本质上是 velocity / higher-order state，随机性更早混进这些额外状态而不是只混进 `x_t`；但现有 defense 证据没有给出 `DDPM/CIFAR10` 上清晰的 `1.2x / 1.5x / 2x` 开销答案，所以不能把它包装成当前主方案。

**文献锚点**

DynaNoise 支持“自适应噪声比固定噪声更合理”的总方向；ReDiffuse 明确表明 repeated queries + averaging 能恢复攻击能力；而 higher-order Langevin 论文引入了 velocity / acceleration 一类的 auxiliary states，但当前公开结果主要不在你这条 unconditional image DDPM 主面上，所以它更像研究线而不是 round-2 bounded successor。另一个有用的事实是，早期 diffusion MIA 工作已经指出 **intermediate timesteps** 往往更脆弱，所以从 timestep separability 反推噪声插位是合理的。([arXiv][3])

## 3) Evaluation protocol

### 报告标准

主指标顺序固定为：

1. **TPR@1%FPR**
2. **TPR@0.1%FPR**
3. **AUC**
4. **ASR**

`64 → 128 → 256 → 512` 的 packet ladder 不变。统计上用 **1000 次 bootstrap**，报 `mean ± std`，至少给 `TPR@1%FPR` 的 `95% CI`；方法间比较用 paired test，并对多重比较做校正。你自己的文档已经把“低 FPR 优先于 AUC”写得很清楚，所以这轮不能再让 `AUC-only win` 混进 promotion。    

### 三块评估板，避免 concealment

对 **H1/H2**，每次都同时报三块板：

* **forgotten subset board**：被编辑的 members vs 匹配 non-members
* **retained subset board**：未编辑 members vs 匹配 non-members
* **full split board**：原始整体评估面

只有 forgotten subset 降而 retained/full 不看，最容易把 concealment 当 success。至少要看到：

* forgotten subset 下降最明显
* retained subset 不出现反向恶化
* full split 在主指标上仍有方向性改善
* defense-aware retrained attacker 下仍保留下降。([arXiv][5])

### Adaptive / defense-aware attacker

**H1/H2：必须 defense-aware shadow retraining。**

* **何时做：**候选线通过 `128` packet 后做一次，不需要在 `64` 就把整套重攻击全开。
* **shadow 是否带 defense：**要。shadow 必须带**同一 defense**，否则只是在测 threshold transfer。
* **attack model 是否重训：**要。对参数改变型 defense，**重训攻击头**是 mandatory；只调 threshold 只能当 diagnostic。
* **shadow 数据：**与 target 训练数据分离；bounded pilot 可先用与 target packet 同量级的 shadow member / non-member 对，promotion 后再放大。

这和你上传的 adaptive-attacker memo 是一致的：重复查询、defense-aware retraining、mean/variance 都要进来。

**H3 / 所有随机防御：必须 repeated-query aggregation。**

* **3/5/10 repeats 是什么：**同一个 query，同一个 `x_0` 或同一个输入样本，换不同随机种子，独立跑 `k` 次。
* **聚合方式：**三种都测，攻击者取最强：

  * `s_mean = mean(s_1,...,s_k)`
  * `s_max = max(s_1,...,s_k)`
  * `s_var = mean(s) + β * std(s)`，`β ∈ {0.5, 1.0}`
* **SimA 怎么聚合：**先对每次 repeat 单独算 denoiser-norm score，再聚合这些 score；不要先平均 `ε` 再算一次。
* **variation-style attack：**如果接口暴露多次随机输出，就让攻击者拿到至少 mean；如果 variance 也可见，就把 variance 也给攻击者。

ReDiffuse 的核心结论就是 averaging multiple stochastic outputs 会恢复 black-box membership signal，所以随机防御不用 repeated-query 审查，结论不可信。([arXiv][6])

### Budget-fixed fairness

**同一个总 query budget 下比。**

* 防御用 `k=10` repeats 时，baseline 也按 **相同预算**记账。
* 最公平的做法有两种，二选一：

  1. **固定每个样本预算**：baseline 与 defense 都允许 `k` 次调用
  2. **固定总预算 `B`**：两者都只在 `floor(B/k)` 个样本上评估
* 不接受“baseline 1 次，防御 10 次聚合”这种不对称比较。

这是你 adaptive memo 里“keep budget fixed”的直接落地。

### Attack suite 配置

* **PIA / GSA**：当前主锚点，不变。 
* **DRC**：bounded second-family challenger。最接近 paper-faithful 的 first setting 是 **deterministic DDIM restoration**、**interval = 5**、**语义相似度用 CLIP image encoder cosine**，CIFAR 侧用 **DINO attention mask top 50% ROI**。([arXiv][7])
* **SimA**：按官方实现，DDPM 配置里有 `attack_num=20`、`interval=10` 的默认设置；概念上它是 denoiser-norm based attack，所以你的 in-repo 轻量 challenger 可以先从这个 norm-based scorer 开始。([arXiv][8])
* **variation-API / averaging**：paper-faithful 第一档就是 **10 次 variation 取平均**。这正好可以直接借来做 stochastic defense 的黑盒 adaptive audit。([arXiv][6])
* **CLiD**：当前 unconditional `DDPM/CIFAR10` **不做**。以后若扩展到 class-conditional，再单独立项。

## 4) Kill gates

**H1 kill gate**

* 到 `128` packet、总计约 `<=2 GPUh` 时，若 **full split 的 `TPR@1%FPR` 或 `TPR@0.1%FPR` 相对 `stochastic-dropout` 改善不到 5%**，停。
* 若只有 forgotten subset 降，但 retained/full 平或反向，停。
* 若 defense-aware retrained attacker 能恢复掉 **80% 以上** 的名义 gain，停。
   

**H2 kill gate**

* `64` scout 上训练不稳，或需要 `r>8`、`>6000` step 才刚看到弱信号，停。
* 若只改 `AUC`，不改 low-FPR tail，停。
* 若 proxy attack head 极度不稳、必须频繁 refresh cache 才工作，停。
   

**H3 kill gate**

* `k=3` repeated-query 就把 gain 基本抹平，停。
* 需要 `σ` 大到明显伤 utility 才超过 dropout，停。
* 一旦发现这条线必须转向 higher-order Langevin 才能继续，停；那已经超出本轮 bounded 范围。([arXiv][6])

## 5) Engineering risk

**H1 最大坑**

* forget-set ranking 很容易过拟合当前攻击族；所以我建议 rank aggregation，而不是只吃一个 scorer。
* SISS 真正 paper-faithful 实现比“retain - forget”稍复杂，但这不是第一轮必须跨过的门槛。
* concealment 风险是真实的，不是文献礼貌性提醒。 ([arXiv][1])

**H2 最大坑**

* MP/SMP-LoRA 论文主要在 latent-diffusion attention 上讲得最清楚；迁到 unconditional DDPM 需要你自己选插位。
* proxy MIA head 是这条线最可能不稳的部分，比 rank 更敏感。
* cache refresh 如果太频繁，会把“轻量 defense”重新做成重训练。 ([arXiv][9])

**H3 最大坑**

* easiest to implement，也 easiest to break。
* repeated-query 攻击是这条线的天然克星。
* higher-order Langevin 的 exact image-DDPM overhead 现在不够清楚，不能拿来做算力预算承诺。([arXiv][6])

**H4/H5 风险**

* H4 不是没路，而是太容易变成“更重、未必更私有”的分支。
* H5 则直接把问题换成架构升级，不适合当前 admitted 面。([arXiv][4])

## 6) Minimal bounded pilot

我建议的最小包只有四步：

**Step 0 — CPU prep**

* 聚合 `GSA + PIA` 风险分数
* 产出 `k=16/32/64` forget list
* 从现有 `PIA` traces 计算 timestep separability，给 H3 备用
   

**Step 1 — H1 scout**

* `k=32`
* `λ=0.5, α=0.5, lr=1e-5`
* `1000` steps
* `64` packet
  预算：约 **0.75–1.0 GPUh**。 

**Step 2 — H2 scout**

* middle + last-2-up blocks
* `r=4`
* `lr=5e-5`
* warmup `500` + total `3000` steps
* frozen cache
* `64` packet
  预算：约 **1.0–1.5 GPUh**。 

**Step 3 — Promote only the winner**

* 只让 H1/H2 中更强的一条升到 `128`
* 同时开一次 defense-aware shadow retraining
* 对 stochastic variant 额外跑 `k=3/5` repeated-query audit
  预算：约 **1.0–1.5 GPUh**。 

**Step 4 — Optional H3 sidecar**

* 只在还剩余预算时做
* `top-4` vulnerable timesteps
* `σ` 扫 `0.01 / 0.02 / 0.05 * std(ε_t)`
* 只做 inference-side audit，不开新训练
  预算：应控制在 **剩余边角预算** 内。 

这样整包可以压在 **约 `3–4 GPUh`** 里，符合你给的 `<=4h GPU on RTX 4070-class` 优先边界。成功标准也不复杂：**在 adaptive review 下，`TPR@1%FPR` 或 `TPR@0.1%FPR` 相对 `stochastic-dropout` 至少再降 5%，且没有明显 utility collapse。**  

## 最终判断

这轮最诚实的动作不是再扩 shortlist，而是：

1. **先做 H1**
2. **并行保留 H2**
3. **把 H3 限定成 cheap stochastic audit**
4. **把 H4/H5 从当前主槽位拿掉**

也就是说，本轮真正值得实现的 defense successor 不是“再来一个泛泛轻量 trick”，而是 **一个 tail-targeted post-training edit（H1）+ 一个 lightweight objective-shaping backup（H2）**。这两条线是目

前唯一既尊重你上传包里的治理边界、又在外部文献里有足够 anchor、还能在当前 admitted `DDPM/CIFAR10` 面上用小预算做出 honest 结论的组合。  ([arXiv][1])

我先给判断：联网核对完 2024–2025 的 primary sources 后，你这包材料里的核心 blocker 基本是对的——我没有看到一个已经 **decision-grade、single-image、query-only、且 clearly outside recon/variation** 的“第四家族”。当前最强的 diffusion black-box 结果，还是围绕 image-to-image probing：REDIFFUSE 是 repeated variation + averaging；GenAI Confessions 是 strength sweep + distance vector + logistic regression；NDSS 2025 的 fine-tuned diffusion 黑盒线本质上也是 reconstruction/scores 框架；而 CDI 的答案则是直接承认单图往往不够稳，转向 **set-level aggregation + statistical testing**。([Proceedings of Machine Learning Research][1])

放到你当前 repo 语境里，这意味着目标仍然应该是：在固定的 `DDPM/CIFAR10`、`final-output/query-only`、`<=4h GPU`、`low-FPR 优先` 约束下，优先找 **新的 readout / aggregation**，而不是再包装一个 recon 小变体。你当前 surface、baseline、low-FPR 优先级、4070 级预算和 64/128→256/512 的 promotion 纪律，我都按你文件里的合同来读。    

## 1. 重新排序

**按执行优先级**，我会改成：

1. **H1 Response-cloud geometry**
2. **H5 Micro-bag statistical audit**
3. **H3 Frequency-sliced stability probe**
4. **H2 Strength-response curve**
5. **H4 Counterfactual patch-recovery**

和你现有 deep-analysis 的差别只有一处：我会把 **H5 提到 H3 前面**。原因不是 H5 更像“新 primitive”，恰恰相反，H5 更像 **aggregation wrapper**；但你这里主指标是 low-FPR，而 CDI 最强的结论正是“单图 MIA 常常不够强，真正可辩护的是 selective aggregation + rigorous testing”。相反，2025 的频域论文把自己的贡献明说成可无缝插入现有攻击、几乎不增时间成本的 **plug-in module**，这更像 enhancer，不像已经成立的独立 family。 ([CVF开放获取][2])

如果你只按“**纯 sample-level signal novelty**”排，不看 low-FPR 执行价值，那我会写成：

**H1 > H3 > H2 > H4**，而 **H5 单列为 aggregation/evidence wrapper**。
因为 H2 已经非常接近 GenAI Confessions 的 strength-vector 路线；H4 虽然概念上最新，但我没找到 2024–2025 diffusion-image primary source 能把 patch-recovery 支撑到 decision-grade。最接近的正证据只是结构记忆论文里“members 的结构更稳、对 distortions 更鲁棒”，这还不是 local completion。([arXiv][3])

## 2. 最值得先做的两条

**如果你问的是“最该先做的两条执行线”**：
我选 **H1 + H5**。

* **H1** 是目前最像“genuinely different sample-level family”的候选。它把 readout 从 `original→output` 改成 `output→output cloud`，即看 attraction basin 的紧致度、维度和熵，而不是看重建误差本身。REDIFFUSE 和 GenAI Confessions 都给了它足够强的动机，但还没有把 cloud geometry 这套 readout 做满。([Proceedings of Machine Learning Research][1])
* **H5** 不是新 primitive，但它最符合你现在的 **low-FPR honesty** 目标。CDI 的核心结论不是“某个单图分数突然很强”，而是“弱信号经过 selective aggregation 和统计检验，才变得可辩护”。对你这个 repo 来说，H5 最像把弱单图 probe 变成低误报证据的桥。([CVF开放获取][2])

**如果你坚持“只做两条纯 signal family”**：
那我选 **H1 + H3**。
但我会把 H3 明确写成 **orthogonal enhancer candidate**，不是独立主线。频域论文本身就把它定位为可插拔增强器；再叠加 `32x32 CIFAR10` 的分辨率限制，我不建议把 H3 当第一独立 pilot。([arXiv][4])

## 3. 每条 hypothesis 的最小实现包

### H1 — Response-cloud geometry

**Query design**：固定两个 timestep，先用中噪声区间的双点包，例如 `t={150,250}`，每个 4 次重复，共 **8 queries / image**。不要把 primary score 设计成 `seed↔output` 距离；同一批输出同时留给 budget-matched recon comparator 即可。REDIFFUSE 和 GenAI 都说明 repeated image-to-image probing 会让 member 输出更贴近、更稳定，但它们的 readout 仍主要落在 seed-anchor 上。([Proceedings of Machine Learning Research][1])

**Aggregation**：先上 CPU-cheap feature stack：
pairwise SSIM / DCT-band distance / shallow embedding cosine；然后取 `mean pairwise distance`、`covariance trace`、`top-eigenvalue ratio`、`1-NN radius`、`entropy proxy`。最后用一个很小的 logistic regressor。这里的 distinctness 在于 **只喂 cloud-shape features**，不把平均 recon distance 放进主 scorer。这个方向和你当前“别把 recon 轻改包装成新 family”的 blocker 是一致的。

**统计检验**：
AUC 用 DeLong；`TPR@1%FPR` 和 `TPR@0.1%FPR` 用 paired bootstrap；同时做一个 **distinctness gate**：看 H1 分数和 budget-matched recon 分数的 Spearman 相关，如果相关极高且 stacked gain 消失，就把它降级为 recon enhancer，而不是新 family。你当前评估协议和 adaptive-attacker 纪律都要求固定预算、报告 tail metrics 和方差。 

**成本**：用你现有 planning assumption 的 `~2s/query`，512 总样本屏幕下，8 queries 约 **2.27h**，12 queries 约 **3.41h**；64/128 pilot 则按比例缩小。这个是最适合“一次 pilot 内同时拿到 candidate 与 budget-matched recon comparator”的线，因为同一批输出可以双用。 

**Kill gate**：
在 **256/512** 上，如果 geometry-only scorer 仍然

1. 对 `TPR@1%FPR` 没有稳定正增益，
2. 与 recon 分数相关性接近塌缩，
3. 特征重要性又回到 seed-anchor 距离，
   那就别再把它当新 family，直接降为 recon/variation readout enhancer。

---

### H2 — Strength-response curve

**Query design**：4 个 strength/timestep 点，2 次重复，共 **8 queries / image**。对 DDPM/CIFAR10，我会用等间隔 timestep 而不是抽象 strength 名义值，例如 `t={80,160,240,320}`。GenAI Confessions 的核心就是多强度 image-to-image 生成、对每个强度重复、再把距离向量交给 logistic regression。([arXiv][3])

**Aggregation**：
每个点取 `min/mean distance`，再提 `slope`、`curvature`、`AUC of curve`、`between-repeat variance`。实现上先别用复杂 spline，直接用二次项或分段线性就够。SeqMIA 虽然不是 diffusion 图像同设定，但它对“序列模式比单点指标更强”的主张，可以给这个方向方法论背书。([GitHub][5])

**统计检验**：
不是拿 H2 去打 recon，而是先拿它打 **same-budget single-strength** baseline。因为如果 full curve 只和一个两点 slope 打平，它本质上就是“贵一点的 variation curve-fitting”。

**成本**：
8–10 queries，对 512 总样本屏幕约 **2.27–2.84h**。

**Kill gate**：
如果 full-curve model 对同预算的 `single-strength` 或 `two-point slope` 没有清晰增益，那就直接把 H2 固定为 **strong baseline / ablation**，不要继续包装成新家族。这个方向已经太接近 GenAI Confessions。([arXiv][3])

---

### H3 — Frequency-sliced stability probe

**Query design**：3 个 spectral views × 3 repeats，共 **9 queries / image**。在 `32×32` 上我建议用 **DCT masks** 而不是 FFT：
低频保留、带通保留、以及高频衰减视图。频域论文的实质是：现有 diffusion MIA 常被高频处理缺陷干扰，因此做 frequency-aware calibration 能改善已有攻击。([arXiv][4])

**Aggregation**：
对每个 band 分别算 `retention`、`pairwise dispersion`、`band gap`；最终只看 `low-vs-mid`、`mid-vs-high` 的差，而不是单个 band 的原始分数。这样更像“频域稳定性 profile”，不是“做了个滤波版 recon”。

**统计检验**：
H3 不该先单打独斗。最小 honest 包是：先做 **H1 baseline**，再测 `H1 + H3` 的 late fusion 是否在 `TPR@1%FPR` 上有独立增益。因为该文自己就把方法定位成 plug-and-play module。([arXiv][4])

**成本**：
9 queries，512 总样本约 **2.56h**。

**Kill gate**：
如果在 256/512 上，`H1 + H3` 相比 H1 单独几乎没有 tail gain，或者 gain 只体现在 AUC 平均值而不体现在 `TPR@1%FPR`，那 H3 就只保留为 optional enhancer，不给主槽位。

---

### H4 — Counterfactual patch-recovery

**Query design**：如果真做，我不会用你旧草案里的 `8×8/16×16` 起步；对 `32×32 CIFAR10` 这太激进。最小包应该是 **4 个 mask pattern × 2 repeats = 8 queries**，从 `4×4` center/random patch、细条纹、checkerboard 开始。理由不是文献给了这个答案，而是你这个 surface 分辨率太低。

**Aggregation**：
只看两类分数：`masked-region recovery` 和 `global coherence drift`。而且 recovery 不能直接和原图全局比，必须有一个 **held-out non-member control distribution**，否则“better than expected”没有基线。

**统计检验**：
先做 one-sided test：member 的 masked recovery 是否显著高于 control mean；再做 bootstrap CI。这里我没有找到 2024–2025 的 diffusion-image primary source 能把这个方向直接支撑到 query-only 单图黑盒审计；最接近的只是结构记忆论文说明 member 的结构更稳、更抗扰动。([arXiv][6])

**成本**：
8 queries，512 总样本约 **2.27h**。

**Kill gate**：
64/128 pilot 上只要出现“方差大于 member/non-member gap”，或者不同 mask pattern 的信号方向互相打架，就直接停。H4 是最该早停的那条。

---

### H5 — Micro-bag statistical audit

**先说定位**：
H5 不是新 physical signal；它是 **evidence wrapper**。这点一定要写清。CDI 的 primary-source 结论是：对大模型，单图 membership signal 往往不够强，所以要靠 dataset-level aggregation + rigorous statistical testing。H5 的价值，是把这个思想压缩成“单图派生出的 micro-bag”，但它本质上仍是准 set-level 证据，不该冒充 sample-level breakthrough。([CVF开放获取][2])

**Query design**：
8 个近身份变体，每个 1 个弱 probe：`±2px crop`、`brightness ±5%`、`one low-pass`、`one band-pass`、`one 4×4 mask`、`one tiny hue shift`。若 bag score 靠近决策边界，再自适应补到 12。这里最现实的 base weak probe 不是 H1，而是一个固定-t 的简单 retention score；因此 H5 更像 **aggregation over weak probes**。这一点和你 prompt 里“必须区分 sample-level evidence 与 set-level evidence”的要求是一致的。

**Aggregation**：
最小 honest 版不要上小 MLP，先上 **bag mean + bag variance + signed-rank / Welch**。你要的是 low-FPR 诚实度，不是另一个难解释的小黑盒。

**统计检验**：
对每个 image 的 micro-bag，先算一个 bag score；再用 held-out non-member calibration bank 给它一个经验 p-value。之后如果真的跨多张图去聚合，那就已经进入 CDI 风格的 dataset inference，不要再称之为 single-image result。([CVF开放获取][2])

**成本**：
8→12 queries，512 总样本约 **2.27→3.41h**。

**Kill gate**：
如果只有把 bag size 拉到 16+、或者大量 adaptive escalation 后 tail 才起来，那它就不符合你当前 `<=4h` 和 bounded-pilot 合同；另外，只要 empirical FPR 明显高于 calibration nominal，就立刻停，这条线最怕“看起来低 FPR，实际没校准”。

## 4. low-FPR 最关键的校准 / 显著性设计

这里我给的是你现在最该锁死的 4 条：

**第一，阈值集和评估集必须分开，而且阈值集要比 pilot 大。**
`TPR@1%FPR` 尤其 `TPR@0.1%FPR` 在 64/128 pilot 上非常脆；如果没有一个锁定的更大 non-member calibration bank，0.1% 尾部基本不诚实。64/128 适合做 direction check，不适合认真吹 0.1% tail。这个和你文件里“low-FPR 优先、bounded pilot 先、再到 256/512”是同一套逻辑。  

**第二，所有 candidate 都必须和 budget-matched recon comparator 共享同一批 outputs。**
这样才能做 paired comparison，也能防止“新方法只是多花了 query”。这点和你的 adaptive-attacker 合同一致：固定预算、允许 repeated-query aggregation、同时报告均值和方差。

**第三，显著性不要只看 AUC。**
AUC 用 DeLong；`TPR@1%FPR`/`TPR@0.1%FPR` 用 paired bootstrap 或 exact binomial interval；如果是 H5 这类 bag-level 线，再加 permutation p-value。你文件里已经明确把 `TPR@1%FPR` 放在 AUC 前面，我建议黑盒线 selection 也按这个顺序读，只把 `AUC≈0.85 recon baseline` 作为 sanity floor。 

**第四，增加一个“distinctness gate”。**
不是所有 gain 都算新 family。最实用的 gate 是：
`corr(candidate, recon)`、`stacked gain over recon`、`same-output ablation`。
如果 candidate 和 recon 高度同排，且叠加后没独立 tail gain，就不要再往“genuinely different”上写。你当前 blocker 文件其实就在逼这个判断。

## 5. Kill gates，我建议再加一个全局版

除了每条自己的 kill gate，我会再加三个 **全局 gate**：

1. **Distinctness gate**：和 budget-matched recon 同排太高、叠加后无独立增益，直接降级为 enhancer。
2. **Tail gate**：只提 AUC、不提 `TPR@1%FPR`，不升。
3. **Bounded-cost gate**：平均 query 超过 12、或者 512 screen 明显冲破 4h，不升。

这三条和你现有 promotion contract 是兼容的：先 bounded pilot，再 256/512 验证，最后才谈 admitted。 

## 6. 如果只能做 1 个 4 小时 pilot

我选 **H1 basic**。

更具体地说，我会这样跑：

* **256+256 总样本**，不是 64+64
* **8 queries / image**，`t={150,250}` 各 4 次
* 只做 **geometry-only features**
* 同时从同一批 outputs 里算一个 **budget-matched recon comparator**
* 主读 `TPR@1%FPR`，AUC 只做 sanity
* 如果前半程 tail 完全没起色，就不再补到 12 queries

我选 H1 而不是 H5/H3/H2/H4 的原因很简单：

* **H1 是唯一最像新 sample-level family 的候选**。([Proceedings of Machine Learning Research][1])
* **它能用同一批输出顺手给你 recon comparator**，所以 distinctness 判断最干净。
* **H5 需要一个 base weak probe**，先做它会把“新 family”和“aggregation wrapper”缠在一起。([CVF开放获取][2])
* **H3 更像 enhancer**，不值得拿 first pilot 主槽位。([arXiv][4])
* **H2 太像已知 strength-vector baseline**。([arXiv][3])
* **H4 在 32×32 上风险过高**。

最后一句压缩成内部 brief 口径就是：

**先做 H1，当“是否真的存在 recon 之外的 sample-level readout”这一问的主判官；H5 作为 low-FPR wrapper 紧随其后；H3 只做 fusion enhancer，不做 first-line standalone；H2 作为强 baseline；H4 暂缓。**
这个排序最贴合你现在的 admitted surface、预算合同和 low-FPR 优先级。

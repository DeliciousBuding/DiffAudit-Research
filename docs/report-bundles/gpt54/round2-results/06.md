结论先放前面：我建议 **`H1 > H2 > H5 > H4 > H3`**。现在就做 `H1`；`H2` 是紧邻 fallback；`H5` 是 honest 的语义 pivot，但不是原语义下的 X-90 解决；`H4/H3` 本轮都不该开。原因很直接：当前 blocker 不是 tri-evidence 想法本身，而是 `PIA=512`、`TMIA-DM=256` 造成的 **512 共享表面缺口**；而 round 2 已经冻结成 **CPU-first bounded review**，不是自动补 GPU 的问题。现有 canary 只证明 256 对齐面有真信号，不足以跳过 512 身份一致性。   

外部方法依据也支持这个排序。ICML 2024 的 diffusion QR attack 证明：可以用 quantile regression 加“小而弱的 bagged attacker”替代同架构 shadow 训练，而且在该论文里比先前方法更省算力、效果更强；ICML 2024 的 RMIA 说明低-FPR 能力不必绑定大量 reference，甚至在 reference 很少时也仍然可用；CVPR 2025 的 CDI 则明确表明，当 pointwise diffusion MIA 不够强时，dataset-level aggregation 是有效替代，但那是**改语义**；NeurIPS 2025 poster 的 SHAPOOL 说明共享 shadow pool 可以降成本，但它仍然是 shadow-family 方案，不适合你这轮的 CPU-first bounded review。([Proceedings of Machine Learning Research][1])

## 1. Ranked recommendation

**Do now — `H1: Temporal QR surrogate calibrated on TMIA-DM-256`**
这是最适合当前仓库的一条，因为它把第三证据直接建在 **同一批 512 IDs** 上，天然消掉 identity mismatch；同时它又承认 `TMIA-DM` 目前只有 256 teacher surface，所以先在 256 上做 teacher-calibrated validation，再 frozen transfer 到 512。它符合当前 gray-box threat model，因为只需要 target 侧的时间/残差/epsilon 轨迹，不需要梯度，也不需要新 shadow 家族。  

**Immediate fallback — `H2: RMIA / few-reference temporal LR`**
如果 `H1` 的问题不是“没信号”，而是“多特征 surrogate 在 tail calibration 上不稳”，就转到 `H2`。我把 `H2` 理解成更结构化的 few-reference / LR family：语义更干净，但默认不比 `H1` 更稳，因为它把成败压到“单个 temporal statistic 选得对不对、null 设得准不准”这两件事上。RMIA 的公开证据支持它是合格 fallback，但不支持它优先于 `H1`。 ([Proceedings of Machine Learning Research][2])

**Governance fallback — `H5: CDI-style set-level evidence`**
`H5` 是“不要再硬保 per-sample tri-evidence 语义”的最诚实退路。它不是 X-90 原问题的直接解，而是把 milestone 从 “per-sample aligned third signal” 改写成 “set-level bounded evidence of data use”。如果 `H1/H2` 都失败，我建议认真考虑 `H5`，但在文档里必须明写：**这是重写目标，不是假装原目标被解决**。   ([CVF开放获取][3])

**Do not do now — `H4: Shared shadow pool + bootstrap`**
这条不是坏路线，但它超出这轮边界。你的成本模型、硬件约束、CPU-first 哲学都已经把当前窗口限定在 “≤4h GPU on RTX 4070-class，先 CPU 再确认”，而内部 deep analysis 也把 `H4` 估成 “tens to low hundreds of GPU-hours”。SHAPOOL 说明它值得保留为更后面的 shadow-faithful 方案，但不是 round 2。    ([OpenReview][4])

**Do not do now — `H3: Perturbation / feature-density temporal signal`**
这条最大问题不是算力，而是**冗余风险**。`PIA` 已经是 trajectory-consistency anchor，`H3` 很容易只是换个 perturbation protocol 再测一次同类信号；再加上 adaptive attacker 协议要求 repeated-query aggregation 和 fixed budget，round 2 不值得先把复杂度加在这里。它更像 side experiment，不像 blocker-resolution 主线。  

## 2. Top candidate implementation spec

我建议把 `H1` 明确写成 **teacher-calibrated temporal surrogate**，而不是“做一个新 classifier 试试看”。teacher 是 `TMIA-DM-256`，student 是一个 **target-only temporal score**；student 只能在 256 teacher surface 上拟合和校准，之后必须冻结，再转移到 512 IDs。这样才不会把 blocker-resolution 和“发明新攻击”混在一起。 

**Temporal feature 选取**
第一版不要吃整条 raw trajectory，也不要做 adaptive step selection。用一个小而稳的 summary bank 就够了，分四类：

1. **level**：early / mid / late 三段的 reconstruction-loss mean，以及 residual/epsilon norm mean；
2. **contrast**：early/late ratio、late-minus-early delta；
3. **shape**：全局 slope、late-window slope、二次 curvature；
4. **volatility**：variance / IQR、最大相邻 step jump。
   这套 feature bank 已经覆盖 prompt 里问到的 loss trajectory、residual norm、early/late ratio、slope、curvature、variance，但维度仍够低，不会在 256 teacher surface 上过拟合。gray-box threat model 也允许这样做，因为你有中间输出/epsilon 级信号，不需要白盒梯度。 

**Timestep grid**
先用 **固定、轻微 late-biased 的 12–16 点 grid**，不要 adaptive sampling。原因不是“adaptive 一定差”，而是它太容易把 256 teacher surface 调成假稳定。round 2 的目标是 honest transfer，不是局部榨干 teacher board。固定 grid 才能让 64/128/256/512 四个 packet size 真正可比。 

**QR family**
默认用 **8–16 个 bagged、regularized linear quantile heads**。我不建议一上来就用 GBDT：256 teacher surface 太小，tree-based quantile model 很容易做出看起来很锋利、实际上不稳定的 tail。线性 QR + bagging 的好处是：弱头足够便宜、可解释、方差低，且与 diffusion QR 论文的“bag of weak attackers”方向一致。只有当 OOF rank 明显 underfit、但稳定性仍好时，才加一个非常浅的树模型做对照，不做默认主线。([Proceedings of Machine Learning Research][1])

**Calibration**
默认用 **out-of-fold isotonic calibration**，把 raw surrogate score 映射到 `TMIA-DM-256` 的 teacher scale。`quantile matching` 只做锁定后的 fallback：当 isotonic 明显把 top 1–5% tail 压扁时才换，而且不能在同一 validation 上来回挑。我的判断是：在 256 teacher surface 这么小的情况下，先要 monotone 与稳定，再谈 tail 美观。

**Scoring pipeline**
我建议固定成下面这条：
`F(x)` → bagged QR raw score `s_H1(x)` → OOF isotonic 得到 `c_H1(x)` → 用 256 calibration stats 做 robust z-score → **只替换 tri-evidence 里的 TMIA-DM 分量**，保留 `PIA + CDI + fusion rule` 不变。
fusion rule 也不要重调权重；直接沿用现有 canary 的 scalar contract（例如 robust z-score sum / 等权融合家族）最诚实，因为这次要解决的是“第三信号缺口”，不是顺便把 fusion 再重新最优化一遍。现有 256 canary 已经证明该 fusion family 有真信号，但还不能跳过 512 consistency。 

## 3. Validation protocol

我建议分四段跑，但只让前两段决定是否继续。

**P0 — 64-sample scout**
目标只是确认 extraction 与 feature bank 正常：没有 NaN、没有塌缩、early/mid/late summary 有区分度。这个阶段不做任何 promotion claim。 

**P1 — 128-sample intermediate calibration**
在 128 上只做两件事：锁 feature bank，锁 calibrator 选择（默认 isotonic）。任何需要在 128 上频繁来回试的复杂模型都该被淘汰，因为这说明方法本身依赖 tuning。 

**P2 — 256-sample teacher validation**
真正的 hard validation 要在完整 `TMIA-DM-256` teacher surface 上做，用 repeated OOF / bootstrap，而不是在 128 上拍脑袋定胜负。这里要比较 surrogate 与 `TMIA-DM-256` 的：

* Spearman / Pearson
* AUC
* TPR@1%FPR
* TPR@0.1%FPR
* threshold CV
  同时还要比较 `PIA + CDI + H1` 是否真的优于 `PIA + CDI`。low-FPR 是主指标，不允许只看 AUC 漂亮。   

**P3 — frozen transfer to 512 IDs**
只要进 512，就必须满足三条：

1. 不再改 feature、不再改 timestep grid、不再改 calibrator；
2. 不在 512 上重做任何 calibration；
3. 只报告 frozen transfer 的四个标准指标与 95% CI。
   我还建议补一个很便宜的 shift audit：比较 256 与 512 上 feature summary 的分布是否严重漂移。若漂移太大，512 结果最多能写成 bounded negative/uncertain transfer，不能写成 honest replacement。  

## 4. Promotion gates

**Hard gates**
这些一条不过，就不该继续写成 blocker resolved：

* 不引入新的 identity mismatch；
* 在 `TMIA-DM-256` 上 `Spearman` 和 `Pearson` 都 > `0.8`；
* `|ΔAUC| < 0.05`；
* `|ΔTPR@1%FPR| < 0.05`；
* threshold stability `CV < 0.15`；
* `PIA + CDI + H1` 相比 `PIA + CDI` 在 low-FPR 上有真实增益；我建议把这条锁成 **`ΔTPR@1%FPR > 0.03` 或至少 95% CI 不跨 0**；
* 512 只能 frozen transfer，禁止 512 上二次调参。
  前五条直接来自冻结 gate；最后两条是为了防止“第三证据”其实只是重复测 `PIA` 或只靠 512 反调。  

**Soft gates**
这些不过不一定立刻 kill，但不能 admitted：

* `TPR@0.1%FPR` 不应明显恶化；
* ASR 至少不应与 headline 相矛盾；
* 64→128→256→512 的 packet-size 变化下，headline 方向应稳定；
* 计算成本应明显低于“继续硬补 TMIA-DM 512”；
* 满足 repo 的 full-eval / CI / significance reporting 习惯。   

## 5. Failure modes

下面这些一出现，我会把 `H1` 判成“不可信 surrogate”，而不是“再多调一调”：

第一，**AUC 好看但 low-FPR tail 掉了**。当前仓库已经明确 low-FPR 比 AUC 更重要；如果 `TPR@1%FPR` 对 teacher 不保真，`H1` 就没有解决 blocker。 

第二，**只有在 adaptive timestep selection 或更复杂模型下才有效**。这通常说明你在吃 256 teacher surface 的局部纹理，不是学到了可 transfer 的 third signal。

第三，**surrogate 与 `PIA` 过度共线**。如果 `H1` 单独看还行，但加进 `PIA + CDI` 后 low-FPR 没增益，或者相关性高到只是在重复 `PIA`，那它不配当第三证据。 

第四，**512 improvement 只在 touching 512 之后才出现**。一旦 feature、calibrator、threshold 在 512 上被重调，transfer honesty 就没了。

**何时 pivot 到 `H2`**
当 `H1` 的问题是“有 rank，但 calibration 不稳”，比如 Spearman 还行但 Pearson / tail / CV 过不了，我会转 `H2`。`H2` 的 temporal statistic 我建议先用 **late-window mean reconstruction loss**，把 **early/late ratio** 作为 sensitivity check；reference 先用 **analytical null**，只有当 null 明显失配 tail 时才加 `1–2` 个 lightweight references。它不是天然更稳，只是当 `H1` 的多特征模型过灵活时，`H2` 会更干净。 ([Proceedings of Machine Learning Research][2])

**何时 pivot 到 `H5`**
当 `H1/H2` 都不能在 **identity alignment + low-FPR honesty + real incremental value** 三件事上同时过关时，就该承认：per-sample 语义不值得继续硬保。那时 `H5` 是合理重写，不是失败掩饰。CDI 的公开结果也支持 dataset-level aggregation 可以形成强证据，但那是另一种 claim。  ([CVF开放获取][3])

## 6. Cost and hardware

按冻结上下文，这轮的正确姿势不是“补更多 shadow”，而是 **CPU-first, bounded, no-new-gpu-question**。你们自己的成本和硬件约束已经写得很清楚：4070 8GB、pilot `<= 4h GPU`、CPU 约便宜 10 倍，而且 packet 逻辑就是 64/128 scout 与 calibration，256 validation，512 full eval。综合进度也明确说当前 `active_gpu_question = none`。   

在这个约束下，`H1` 是唯一真正匹配窗口的方法：

* **训练与 calibration** 都可以纯 CPU；
* **feature extraction** 若已有 per-step trace，可继续纯 CPU；
* 若没有 trace，只需对 target DDPM 做一轮有界 batched forward，通常也比新训 shadow 轻得多。
  我会把 GPU 预算写成：`256` teacher validation 最多 `1–2h`，`512` frozen transfer 最多补到 `<=4h`；若 64/128 阶段已经失败，就不应上 GPU confirmation。这个用法符合你们“CPU 校准、GPU 只做确认”的哲学。  

它比“继续硬补 `TMIA-DM 512`”更诚实，不只是因为省钱，而是因为**语义更对**：

* `TMIA-DM` 现在真正可信的 teacher 面只有 256；
* `H1` 先承认这点，再在 teacher 面上验证 surrogate 是否保住 rank/tail；
* 通过后才 frozen transfer 到你真正拥有的 512 IDs。
  这比一上来把大量 GPU 花在“维持旧 shadow 语义”上更符合当前仓库强调的 comparability、contract、low-FPR honesty。   

## 7. Minimal bounded pilot

最小实验包我建议这样锁：

**所需资产**

* 冻结的 `TMIA-DM-256` aligned teacher board：ID、member/nonmember label、`TMIA-DM` score；
* 同一批 ID 上的 `PIA` 与 `CDI` 分数；
* 冻结的 512 evaluation IDs 及其 `PIA/CDI` 分数与 membership labels；
* target DDPM 的 per-timestep scoring/extraction 路径，至少能导出 loss 与 residual/epsilon norm；
* 现有四指标 + bootstrap CI evaluator。   

**最小执行顺序**

1. `64-sample scout`：只检查 feature extraction、feature stability、QR 训练是否正常；
2. `128-sample calibration rung`：锁 feature bank、锁 timestep grid、锁 calibrator；
3. `256-sample teacher validation`：跑 hard gates，拿 OOF rank/tail/stability；
4. 只有 hard gates 全过，才做 `512 frozen transfer`；
5. 512 上只报告四指标、95% CI、与 `PIA + CDI` 的差分。 

**提前收手条件**

* 64 上 extraction 不稳或多数 feature 退化；
* 128 上仍需要 adaptive grid / 复杂模型才能出信号；
* 256 上任一 hard gate 失败。
  一旦出现这些情况，就不要继续给 `H1` 加 GPU；直接切 `H2`，或者若 `H2` 也失败，就把结论诚实写成“per-sample X-90 不值得继续硬保，建议转 `H5`”。 

一句话收束：**本轮最值得继续的是 `H1`，但必须把它实现成 `TMIA-DM-256 teacher-calibrated, frozen-transfer temporal surrogate`；`H2` 作为紧邻 fallback，`H5` 作为语义重写的 honest 退路，`H4/H3` 这轮都不该开。** 这条路最符合当前仓库、当前算力和当前证据治理。

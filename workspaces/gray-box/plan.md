# 灰盒方向初始计划

## 方向定位

灰盒方向关注：

- 只能看到部分内部信息
- 既不像黑盒那么弱，也不像白盒那样完全可见
- 更适合研究“部分可观测条件下”的攻击可行性

## 当前建议工作方式

灰盒方向先不要急着写执行代码，先把攻击设定讲清楚：

1. 允许看到哪些中间输出
2. 不允许看到哪些内部参数
3. 与黑盒 / 白盒相比，灰盒到底增强了什么

## 第一周目标

1. 补一份灰盒方向论文候选列表
2. 定义灰盒方向的最小实验设定
3. 给出所需资产和可能的接口需求

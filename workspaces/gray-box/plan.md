# 灰盒方向计划

## 推荐论文

- `SecMI_ICML2023.pdf`
- `PIA_ICLR2024.pdf`
- `CLiD_NeurIPS2024.pdf`

## 当前定义建议

灰盒 = 可访问部分中间扩散信息、噪声预测、条件相关评分或中间去噪结果，但不可访问完整参数和梯度。

## 起步方案

1. 先把灰盒边界定义写清楚
2. 先复现 `SecMI`
3. 再用 `PIA` 做效率和攻击代价对照
4. 用 `CLiD` 作为更贴近 text-to-image / conditional setting 的扩展参考

## 当前阻塞项

- 需要统一“灰盒”到底允许哪些内部信息
- 需要确定后续系统里灰盒接口如何表达
- 需要区分灰盒和黑盒条件信息利用的边界
